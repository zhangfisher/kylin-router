<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Remote Route Config → Router | 远程加载的路由配置不受信任，可能包含恶意内容 |
| User Input → Dynamic Routes | 动态添加的路由配置可能包含无效或恶意模式 |
| Default Redirect → Navigation | 默认重定向可能形成循环，导致 DoS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | remote routes | mitigate | 验证远程路由配置的数字签名（可选）；使用 HTTPS 加载远程配置；验证响应格式 |
| T-03-02 | Tampering | dynamic routes | mitigate | 验证路由配置格式，拒绝无效配置；冻结路由对象防止运行时修改 |
| T-03-03 | Repudiation | route changes | accept | 路由变更通过路由表可审计，无需额外处理 |
| T-03-04 | Information Disclosure | error messages | mitigate | 远程加载失败时不泄露敏感 URL 信息；生产环境隐藏详细错误堆栈 |
| T-03-05 | Denial of Service | redirect loop | mitigate | 检测循环重定向，设置最大重定向次数（MAX_REDIRECTS = 10） |
| T-03-06 | Elevation of Privilege | remote code execution | mitigate | 验证远程配置仅包含数据而非代码；拒绝包含 eval、Function 等危险配置 |

所有威胁都需要缓解措施，因为动态路由和远程配置是关键安全边界。
</threat_model>
