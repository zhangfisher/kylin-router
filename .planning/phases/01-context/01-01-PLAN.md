<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL → Router | 不受信任的用户输入（URL 路径）进入路由匹配系统 |
| Route Config → Router | 应用层提供的路由配置可能包含恶意模式 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | route matching | mitigate | 验证路由配置格式，拒绝无效配置；使用白名单验证路由模式中的特殊字符 |
| T-01-02 | Tampering | route config | mitigate | 冻结路由配置对象，防止运行时修改；使用 Object.freeze() 保护核心路由表 |
| T-01-03 | Repudiation | navigation history | accept | 浏览器 History API 已提供审计功能，无需额外处理 |
| T-01-04 | Information Disclosure | error messages | mitigate | 路由匹配错误不泄露敏感路径信息；生产环境隐藏详细错误堆栈 |
| T-01-05 | Denial of Service | ReDoS in path patterns | mitigate | 限制路由模式复杂度，拒绝嵌套量词和回溯攻击模式；设置匹配超时（100ms） |
| T-01-06 | Elevation of Privilege | path traversal | mitigate | 规范化路径，移除 ../ 序列；限制路径字符集为 [a-zA-Z0-9-_/] |

所有威胁都需要缓解措施，因为路由匹配是核心安全边界。
</threat_model>
