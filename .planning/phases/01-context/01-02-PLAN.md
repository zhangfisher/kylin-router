完善 src/router.ts 中的编程式导航 API：

   - 使用 this.history.push(path) 导航到新路径
   - 自动触发 onRouteUpdate 回调（通过 history.listen）
   - 支持相对路径和绝对路径
   - 按照 CONTEXT.md 决策 D-28 实现路径处理

2. **replace(path: string) 方法**
   - 使用 this.history.replace(path) 替换当前历史记录
   - 不增加历史记录栈长度
   - 触发 onRouteUpdate 回调

3. **back() 方法**
   - 使用 this.history.back() 返回上一页
   - 等价于 go(-1)

4. **forward() 方法**
   - 使用 this.history.forward() 前进到下一页
   - 等价于 go(1)

5. **go(delta: number) 方法**
   - 使用 this.history.go(delta) 跳转指定步数
   - 支持正数（前进）和负数（后退）

6. **导航状态管理**
   - 添加 isNavigating: boolean 属性（是否正在导航）
   - 在 onRouteUpdate 开始时设置 isNavigating = true
   - 在 onRouteUpdate 完成时设置 isNavigating = false
   - 按照 CONTEXT.md 决策 D-26 实现导航状态 API

7. **导航事件系统**
   - 在 push/replace 开始时触发 'navigation-start' 事件
   - 在 onRouteUpdate 完成时触发 'navigation-end' 事件
   - 事件包含 location 和 navigationType（push/replace/pop）

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User Input → Navigation | 用户点击链接或调用导航方法，不受信任的输入进入导航系统 |
| External Links → Browser | 外部 URL 可能指向恶意网站 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | kylin-link | mitigate | 验证 to 属性格式，拒绝 javascript: 和 data: 协议；仅允许 http/https/相对路径 |
| T-02-02 | Tampering | navigation history | accept | 浏览器 History API 已保护历史记录完整性，无需额外处理 |
| T-02-03 | Repudiation | navigation actions | accept | 导航动作通过浏览器历史记录可审计，无需额外处理 |
| T-02-04 | Information Disclosure | query parameters | accept | 查询参数通过 URL 传递是预期行为，不泄露敏感信息 |
| T-02-05 | Denial of Service | navigation loop | mitigate | 检测导航循环，设置最大连续导航次数（100 次/分钟） |
| T-02-06 | Elevation of Privilege | external links | mitigate | 添加 rel="noopener noreferrer" 到外部链接，防止 target="_blank" 安全问题 |

所有威胁都需要缓解措施，因为导航是用户交互的核心入口。
</threat_model>
