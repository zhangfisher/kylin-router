**类结构：**
```typescript
export class Render {
    constructor(private router: KylinRouter) {}
    
    // 主渲染方法 - 根据 LoadResult 渲染到 outlet
    async renderToOutlet(loadResult: LoadResult, outlet: HTMLElement, options?: RenderOptions): Promise<void>
    
    // 渲染 lit 模板
    private renderTemplate(template: string | TemplateResult, context: RenderContext, outlet: HTMLElement, mode: RenderMode): void
    
    // 创建渲染上下文
    private createRenderContext(route: RouteItem): RenderContext
    
    // 编译模板字符串为 lit 模板
    private compileTemplate(htmlString: string, context: RenderContext): TemplateResult
}
```

**实现要点：**

1. **renderToOutlet 方法**
   - 检查 loadResult.success
   - 失败：渲染错误组件（如果有）
   - 成功：根据 content 类型分发
     - string（HTML 元素名）：创建元素并渲染
     - TemplateResult：直接渲染
     - string（远程 HTML）：调用 compileTemplate 编译

2. **renderTemplate 方法**
   - 接收模板、上下文、outlet 元素、渲染模式
   - 替换模式：outlet.innerHTML = ''，然后渲染新内容
   - 追加模式：直接追加内容到 outlet
   - 使用 lit 的 render() 函数渲染 TemplateResult

3. **createRenderContext 方法（D-04）**
   - 创建包含 router 和 route 的基础上下文
   - 展开 route.data 的所有字段为局部变量
   ```typescript
   const context: RenderContext = {
       router: this.router,
       route: route,
       ...route.data  // 展开预加载数据
   };
   ```

4. **compileTemplate 方法（D-03, D-09）**
   - 使用 lit/html 的 html`` 模板函数
   - 支持 ${} 插值语法
   - 将 context 中的变量注入模板
   - 处理模板中的表达式求值

**渲染模式处理（D-07, D-08）：**
- 检查 route.renderMode 或 options.mode
- 检查元素是否有 data-outlet-append 属性
- 默认使用替换模式

**并行渲染支持（D-05）：**
- 渲染方法不等待子 outlet 完成
- 立即返回 Promise，允许父 outlet 和子 outlet 并行渲染

使用 lit 的 render() 函数和 html`` 模板函数。

在 Render 类中实现模板变量插值系统：

**方法签名：**
```typescript
private interpolateTemplate(templateString: string, context: RenderContext): TemplateResult
```

**实现要点：**

1. **变量替换策略（D-04）**
   - 识别模板中的 ${variable} 占位符
   - 从 context 中查找变量值
   - 支持嵌套路径：${user.name}、${route.data.userId}

2. **变量查找顺序**
   ```
   1. route.data 展开的局部变量（优先级最高）
   2. route 对象的属性
   3. router 对象的属性和方法
   ```

3. **特殊变量处理**
   - ${router} - router 实例
   - ${route} - 当前路由对象
   - ${params} - 路由参数（快捷方式）
   - ${query} - 查询参数（快捷方式）

4. **安全性**
   - 防止 XSS 攻击：使用 lit 的自动转义
   - 避免任意代码执行：只支持属性访问，不支持表达式

5. **实现方式**
   - 使用正则表达式识别 ${} 占位符
   - 使用 Function 构造函数创建安全的求值环境
   - 或者使用简单的字符串替换（更安全但功能有限）

**示例：**
```html
<!-- 模板 -->
<div>
  <h1>${title}</h1>
  <p>User: ${userId}</p>
  <a href="${router.path}/profile">Profile</a>
</div>

<!-- 上下文 -->
{
  title: 'Hello World',
  userId: 123,
  router: routerInstance,
  route: routeObject
}

<!-- 结果 -->
<div>
  <h1>Hello World</h1>
  <p>User: 123</p>
  <a href="/profile">Profile</a>
</div>
```

使用 lit 的 html`` 模板函数确保自动转义。

2. **实现路由变化处理**
   ```typescript
   private async _handleRouteChange(event: CustomEvent) {
       const { route, router } = event.detail;
       
       // 检查是否匹配当前 outlet 的 path
       if (this.path && !this._matchesPath(route.path)) {
           return;
       }
       
       // 获取组件加载结果
       const loadResult = router.currentComponent;
       if (!loadResult) {
           this._renderLoading();
           return;
       }
       
       // 渲染组件
       await router.render.renderToOutlet(loadResult, this, {
           mode: route.renderMode,
           outlet: this
       });
   }
   ```

3. **路径匹配逻辑**
   ```typescript
   private _matchesPath(routePath: string): boolean {
       // 支持精确匹配和前缀匹配
       if (!this.path) return true;
       return routePath === this.path || routePath.startsWith(this.path + '/');
   }
   ```

4. **添加加载状态渲染**

   private _renderLoading() {

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Spoofing | interpolateTemplate | mitigate | 使用 lit 的自动转义，防止恶意脚本注入，限制变量访问路径 |
| T-03-08 | Tampering | renderToOutlet | accept | 渲染操作在客户端执行，无完整性风险 |
| T-03-09 | Repudiation | renderToOutlet | accept | 渲染操作不涉及不可抵赖性需求 |
| T-03-10 | Information Disclosure | interpolateTemplate | mitigate | 防止通过模板注入泄露敏感信息，限制可访问的属性 |
| T-03-11 | Denial of Service | renderToOutlet | mitigate | 限制渲染内容大小，防止大 DOM 挂起应用，实现渲染超时 |
| T-03-12 | Elevation of Privilege | renderToOutlet | accept | 渲染在浏览器沙箱中执行，无权限提升风险 |

**Additional Security Considerations:**
- 模板变量插值使用 lit 的自动 HTML 转义
- 限制模板中的表达式求值，防止任意代码执行
- 远程 HTML 在 Loader 中已清理，这里不再重复处理
- 渲染大内容时使用文档片段优化性能
