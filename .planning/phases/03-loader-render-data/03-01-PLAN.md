<task type="auto">
  <name>Task 2: 实现 Loader 类的本地组件加载功能</name>
  <files>src/features/loader.ts</files>
  <action>
在 src/features/loader.ts 中实现 Loader 类：

**类结构：**
```typescript
export class Loader {
    constructor(private router: KylinRouter) {}
    
    // 主加载方法 - 根据 component 类型分发到不同加载策略
    async loadComponent(component: string | Function): Promise<LoadResult>
    
    // 本地组件加载 - 处理 HTML 元素名（如 'div'、'my-component'）
    private loadLocalComponent(component: string): LoadResult
    
    // 动态导入加载 - 处理函数形式的组件
    private loadDynamicImport(importFn: Function): Promise<LoadResult>
}
```

**实现要点：**
1. **loadComponent 方法**
   - 检测 component 类型（string vs Function）
   - string 类型：判断是否为 URL（以 http:// 或 https:// 开头）
   - URL → 调用 loadRemoteHTML（Task 3）
   - 非 URL string → 调用 loadLocalComponent
   - Function → 调用 loadDynamicImport

2. **loadLocalComponent 方法**
   - 接收元素名字符串（如 'div'、'span'、'my-component'）
   - 返回包含该元素名的 LoadResult
   - 不需要实际创建元素，由 Render 类负责渲染
   - 格式：{ success: true, content: elementName, error: null }

3. **loadDynamicImport 方法**
   - 接收返回 Promise 的函数（如 () => import('./MyComponent.js')）
   - 调用函数获取模块
   - 提取默认导出或命名导出
   - 处理导入失败的情况

**错误处理：**
- 无效的 component 类型：返回 { success: false, content: null, error: new Error("Invalid component type") }
- 动态导入失败：捕获错误并返回 LoadResult

不使用外部依赖，纯 TypeScript 实现。

  <verify>grep -n "loadLocalComponent\|loadDynamicImport" src/features/loader.ts</verify>

  <name>Task 3: 实现 Loader 类的远程 HTML 加载功能</name>
  <files>src/features/loader.ts</files>
  <action>
在 Loader 类中添加远程 HTML 加载方法：

**方法签名：**
```typescript
private async loadRemoteHTML(url: string, options?: RemoteLoadOptions): Promise<LoadResult>
```

**实现要点：**
1. **Fetch 加载**
   - 使用 fetch API 获取远程 HTML
   - 支持 AbortController 取消请求（D-24）
   - 处理网络错误和超时

2. **智能内容提取（D-02）**
   - 解析 HTML 字符串
   - 检查是否包含完整 HTML 文档（包含 <html> 或 <body> 标签）
     - 有 <body>：提取 body 内部内容
     - 无 <body>：使用整个内容
   - 检查是否存在 data-outlet 属性的元素
     - 有：提取该元素的内容
     - 无：使用提取后的内容

3. **安全性检查（D-15）**
   - 默认模式：移除 <script> 标签和危险属性
   - allowUnsafeHTML=true: 跳过安全检查
   - 使用正则表达式或 DOMParser 清理内容

4. **超时处理**
   - 使用 Promise.race 实现超时机制
   - 超时后返回 { success: false, content: null, error: new Error("Load timeout") }

5. **返回格式**
   - 成功：{ success: true, content: htmlString, error: null }
   - 失败：{ success: false, content: null, error: Error }

**辅助方法：**
- `extractContent(html: string, selector?: string): string` - 提取有效内容
- `sanitizeHTML(html: string, allowUnsafe: boolean): string` - 安全清理

使用原生 fetch API，不引入额外依赖。

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| router → remote URL | 不受信任的远程 HTML 内容 crosses here |
| loader → render | 加载的内容传递给渲染层 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | loadRemoteHTML | mitigate | 验证 URL 格式，只允许 http/https 协议，拒绝 javascript: 等危险协议 |
| T-03-02 | Tampering | loadRemoteHTML | mitigate | 实现内容安全策略（CSP），移除 <script> 标签和事件处理属性，除非 allowUnsafeHTML=true |
| T-03-03 | Repudiation | loadComponent | accept | 加载操作不涉及不可抵赖性需求 |
| T-03-04 | Information Disclosure | loadRemoteHTML | mitigate | 检查 URL 是否包含敏感信息，不在错误消息中泄露 URL 详情 |
| T-03-05 | Denial of Service | loadRemoteHTML | mitigate | 实现超时机制（默认 5 秒），限制加载内容大小（1MB），防止大文件挂起应用 |
| T-03-06 | Elevation of Privilege | loadComponent | accept | 组件加载在浏览器沙箱中执行，无权限提升风险 |

**Additional Security Considerations:**
- 远程 HTML 加载默认开启安全模式，移除危险元素
- allowUnsafeHTML 配置项明确告知安全风险
- 使用 AbortController 防止请求挂起
- 错误消息不暴露内部实现细节
</threat_model>
