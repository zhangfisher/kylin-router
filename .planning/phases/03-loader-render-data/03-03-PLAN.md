在 DataLoader 类中添加重试机制：

**方法签名：**
```typescript
async retryLoad(route: RouteItem, outlet: HTMLElement, config: RetryConfig): Promise<LoadResult>
```

**实现要点（D-14）：**

1. **重试策略配置**
   - 默认：固定重试 3 次，每次间隔 1 秒
   - 自定义：支持配置 max、delay、backoff
   - 支持手动重试：通过错误组件的 retry() 方法

2. **重试逻辑**
   ```typescript
   for (let attempt = 1; attempt <= config.max; attempt++) {
       try {
           const result = await this.router.loader.loadComponent(route.component);
           if (result.success) {
               return result;
           }
       } catch (error) {
           if (attempt === config.max) {
               throw error;
           }
           // 延迟后重试
           await this.delay(config.delay, attempt, config.backoff);
           // 触发重试回调
           config.onRetry?.(attempt, error);
       }
   }
   ```

3. **延迟计算（D-14）**
   - linear: 延迟 = delay * attempt（线性增长）
   - exponential: 延迟 = delay * Math.pow(2, attempt - 1)（指数增长）

4. **AbortController 集成（D-24）**
   - 每次重试使用新的 AbortController
   - 导航竞速时取消所有进行中的重试
   - 清理旧请求，释放资源

5. **与版本号机制集成（D-23）**
   - 重试前检查导航版本号
   - 如果版本号不匹配，丢弃重试结果
   - 防止旧导航的重试污染新导航

**辅助方法：**
```typescript
private delay(ms: number, attempt: number, backoff?: string): Promise<void>
private checkNavVersion(): boolean  // 检查导航版本号
```

**手动重试支持：**
- 错误边界组件可以调用 router.retryLoad() 手动重试
- 提供明确的语义和用户控制

使用 async/await 和 Promise 实现异步重试逻辑。
