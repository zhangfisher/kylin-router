    1. 实现 executeRouteGuards 方法：
       ```typescript
       protected async executeRouteGuards(
         matchedRoutes: MatchedRoute[], // 从外到内排序的匹配路由
         to: RouteItem,
         from: RouteItem,
         router: any,
         guardType: 'beforeEnter' | 'afterLeave'
       ): Promise<boolean | string> {
         // 从外到内执行父 → 子
         for (const matched of matchedRoutes) {
           const route = matched.route;
           const guard = guardType === 'beforeEnter' ? route.beforeEnter : route.afterLeave;

           if (guard) {
             try {
               const result = await guard(to, from);
               if (result === false) return false;
               if (typeof result === 'string') return result;
             } catch (error) {
               console.error(`Route ${guardType} guard error:`, error);
               return false; // 守卫失败，取消导航
             }
           }
         }
         return true;
       }
       ```

    2. 实现嵌套路由的守卫执行顺序（父优先）：
       ```typescript
       protected getOrderedMatchedRoutes(matchedRoutes: MatchedRoute[]): MatchedRoute[] {
         // Phase 1 的路由匹配返回从内到外的顺序
         // 守卫执行需要从外到内（父 → 子）
         return [...matchedRoutes].reverse();
       }
       ```

## 威胁模型

### 信任边界

| Boundary | Description |
|----------|-------------|
| 路由配置 → 守卫系统 | 路由配置中的守卫函数可能包含恶意代码 |
| 嵌套路由 → 守卫执行顺序 | 子路由可能尝试绕过父路由守卫 |

### STRIDE 威胁登记

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Spoofing | 路由配置 | accept | 路由配置由应用开发者提供，信任边界明确 |
| T-02-08 | Tampering | 守卫返回值 | mitigate | 验证守卫返回值类型，拒绝无效值 |
| T-02-09 | Denial of Service | 守卫超时 | mitigate | 30秒超时限制，防止挂起 |
| T-02-10 | Information Disclosure | 错误日志 | mitigate | 错误日志不包含敏感信息（如 token） |
| T-02-11 | Elevation of Privilege | 守卫权限 | mitigate | 守卫无法访问 router 的敏感方法 |
