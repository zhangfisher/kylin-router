# Plan 02-01: 实现全局钩子系统

## 目标

实现路由器的全局钩子管理系统，支持 `beforeEach`、`renderEach`、`afterEach` 三种钩子类型，提供完整的导航生命周期管理。

目的：为路由导航提供在导航前、渲染前、导航后三个时机的拦截和处理能力，支持权限验证、数据预加载、日志记录等场景。

输出：完整的钩子注册、执行、取消和错误处理机制。

       ```typescript
       protected async executeHooks(
         type: HookType,
         to: RouteItem,
         from: RouteItem,
         router: any
       ): Promise<boolean> {
         const hooks = this.hooks[type];
         if (hooks.length === 0) return true;

         for (const hook of hooks) {
           const result = await this.runHook(hook, to, from, router);
           if (result === false) return false;
           if (typeof result === 'string') return result; // 重定向路径
         }
         return true;
       }
       ```

    2. 实现 runHook 方法（处理单个钩子执行，包含超时）：
       ```typescript
       protected async runHook(
         hook: HookFunction,
         to: RouteItem,
         from: RouteItem,
         router: any
       ): Promise<boolean | string> {
         return new Promise((resolve, reject) => {
           const timeout = setTimeout(() => {
             reject(new Error(`Hook timeout after 30000ms`));
           }, 30000);

           const next = (result?: boolean | string) => {
             clearTimeout(timeout);
             resolve(result !== undefined ? result : true);
           };

           try {
             const result = hook(to, from, next, router);
             if (result instanceof Promise) {
               result.then(() => clearTimeout(timeout)).catch(err => {
                 clearTimeout(timeout);
                 reject(err);
               });
             }
           } catch (error) {
             clearTimeout(timeout);
             reject(error);
           }
         });
       }
       ```

    遵循 D-11 (30秒超时) 和 D-13 (错误处理) 的决策

## 威胁模型

### 信任边界

| Boundary | Description |
|----------|-------------|
| 钩子函数 → 路由系统 | 用户代码在钩子中执行，可控制导航流程 |
| 导航重定向 → URL 更新 | 钩子可触发重定向，可能导致循环 |

### STRIDE 威胁登记

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | 钩子函数 | accept | 钩子由应用开发者提供，信任边界明确 |
| T-02-02 | Tampering | 路由参数 | mitigate | 钩子接收不可变的 to/from 对象，next 控制导航 |
| T-02-03 | Repudiation | 钩子执行日志 | mitigate | 所有关键操作自动记录到控制台（D-30） |
| T-02-04 | Information Disclosure | router 参数 | mitigate | 限制 router 暴露的敏感 API，仅提供必要方法 |
| T-02-05 | Denial of Service | 无限循环 | mitigate | 30秒超时（D-11）+ 10次重定向限制（D-16） |
| T-02-06 | Elevation of Privilege | next 回调 | mitigate | next 回调仅控制导航流程，不提供额外权限 |
