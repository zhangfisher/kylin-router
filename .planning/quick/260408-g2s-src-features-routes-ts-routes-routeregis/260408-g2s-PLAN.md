# Quick Task 260408-g2s: Routes 类重构为 RouteRegistry

**创建时间:** 2026-04-08
**任务描述:** 将 src/features/routes.ts 的 Routes 类重命名为 RouteRegistry，在 KylinRouter 中创建 routes: RouteRegistry 属性，移除 mixin 模式，更新相关方法和测试

---

## 任务分析

### 当前状态
- `Routes` 类位于 `src/features/routes.ts`
- `KylinRouter` 通过 `Mixin(Routes)` 继承路由功能
- `Routes` 中的方法直接成为 `KylinRouter` 的方法
- 相关属性：`routes`、`notFound`、`defaultRoute`、`current`、`_redirectCount`

### 目标状态
- `Routes` 类重命名为 `RouteRegistry`
- `KylinRouter` 包含 `routes: RouteRegistry` 实例属性
- 移除 mixin 模式，使用组合模式
- 所有路由相关方法通过 `routes` 实例调用
- 测试文件更新调用方式

---

## 任务分解

### Task 1: 重命名 Routes 类为 RouteRegistry

**文件:** `src/features/routes.ts`

**操作:**
1. 将 `export class Routes` 改为 `export class RouteRegistry`
2. 更新类内引用：所有 `this.routes` 改为 `this.routeRegistry` 或保持 `this.routes`（取决于属性名）
3. 更新导出：在 `src/features/index.ts` 中更新导出

**验证:** 类名已更新，导入导出正确

---

### Task 2: 重构 KylinRouter 使用组合模式

**文件:** `src/router.ts`

**操作:**
1. 移除 `Routes` 从 `Mixin` 调用
2. 移除 `Routes` 从导入列表
3. 添加 `import { RouteRegistry } from "./features/routes"`
4. 添加 `public routes: RouteRegistry` 属性
5. 在构造函数中初始化：
   ```typescript
   this.routes = new RouteRegistry();
   this.routes.initRoutes(...);
   ```
6. 更新所有直接调用路由方法的地方：
   - `this.initRoutes()` → `this.routes.initRoutes()`
   - `this.addRoute()` → `this.routes.addRoute()`
   - `this.removeRoute()` → `this.routes.removeRoute()`
   - `this.loadRemoteRoutes()` → `this.routes.loadRemoteRoutes()`
   - `this._matchCurrentLocation()` → `this.routes._matchCurrentLocation()`
   - `this._matchAndUpdateState()` → `this.routes._matchAndUpdateState()`
   - `this._checkDefaultRedirect()` → `this.routes._checkDefaultRedirect()`
7. 更新属性访问：
   - `this.routes`（路由表数组）→ `this.routes.routes`
   - `this.notFound` → `this.routes.notFound`
   - `this.defaultRoute` → `this.routes.defaultRoute`
   - `this.current` → `this.routes.current`
   - `this._redirectCount` → `this.routes._redirectCount`

**验证:** KylinRouter 编译通过，路由功能正常

---

### Task 3: 更新 RouteRegistry 中的 this 引用

**文件:** `src/features/routes.ts`

**操作:**
更新 `RouteRegistry` 类中所有对 `(this as any).history` 和 `(this as any).push` 的引用，因为这些现在通过参数传递或 router 实例传递。

需要修改的方法：
- `_matchCurrentLocation`: 接收 `history` 和 `push` 方法作为参数
- `_matchAndUpdateState`: 移除对 router 实例的依赖
- `_checkDefaultRedirect`: 接收 `push` 方法作为参数
- `_redirectToDefaultOrNotFound`: 接收 `push` 方法作为参数

**验证:** RouteRegistry 不再依赖 `this` 作为 router 实例

### API 变更
**旧 API:**
```typescript
router.routes // 路由表数组
router.current.route // 当前路由
router.addRoute(route) // 添加路由
```

**新 API:**
```typescript
router.routes.routes // 路由表数组
router.routes.current.route // 当前路由
router.routes.addRoute(route) // 添加路由
```
