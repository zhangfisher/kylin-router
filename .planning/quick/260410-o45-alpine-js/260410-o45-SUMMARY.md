### 2. 创建哈希插值工具函数 ✅

**文件:** [src/utils/hash.ts](../../src/utils/hash.ts) (新建)

**功能:**
- 创建 `generateRouteHash()` 函数
- 实现字符串插值逻辑（复用 data.ts 的 params 工具）
- 特殊字符转义，确保生成的 hash 是有效的 Alpine.js store 名称
- 确保不以数字开头（添加下划线前缀）

**代码示例:**
```typescript
export function generateRouteHash(
    route: RouteItem,
    currentPath: string,
    currentParams: Record<string, string>,
    currentQuery: Record<string, string>
): string {
    const hashPattern = route.hash || "{path}";
    const interpolationVars = {
        path: route.path,
        basepath: route.path,
        url: currentPath,
        timestamp: Date.now(),
        ...currentQuery,
        ...currentParams,
    };
    let hash = params(hashPattern, interpolationVars);
    hash = hash.replace(/[^a-zA-Z0-9_-]/g, '_');
    if (hash && /^[0-9]/.test(hash)) {
        hash = '_' + hash;
    }
    return hash;
}
```

```typescript
async renderView(content, hash, hasData, data?) {
    // 如果启用 keepalive，检查缓存
    if (this.keepalive && this._viewCache.has(hash)) {
        const cachedView = this._viewCache.get(hash);
        this._switchToView(hash, cachedView);
        return;
    }

    // 创建新的视图容器
    const viewContainer = document.createElement('div');
    viewContainer.setAttribute('data-hash', hash);

    // 根据 hasData 决定是否创建 Alpine.data
    if (hasData && data) {
        this.addStore(hash, data);
        viewContainer.setAttribute('x-data', hash);
    }

    // 渲染内容...
}
```

**缓存管理:**
- `keepalive = true`: 视图被缓存，切换时隐藏/显示
- `keepalive = false`: 切换时销毁旧视图和对应的 Alpine.data

## 使用示例

### 基本使用

```typescript
// 路由配置
const routes = [
    {
        name: "user",
        path: "/user/:id",
        view: "user-profile.html",
        data: {
            from: () => fetchUser(params.id)
        },
        hash: "user-{params.id}" // 生成类似 "user-123" 的哈希
    },
    {
        name: "posts",
        path: "/posts",
        view: "posts-list.html",
        data: {
            from: () => fetchPosts()

        hash: "posts-{query.category}" // 生成类似 "posts-tech" 的哈希

// 模板中使用 Alpine.js
```

## 技术细节

### 哈希生成规则

1. **默认值**: 如果未指定 `hash`，使用 `"{path}"`
2. **插值变量**: 支持路径参数、查询参数、系统变量
3. **字符转义**: 非字母数字字符替换为下划线
4. **数字前缀**: 如果以数字开头，添加下划线前缀

### Alpine.js 集成

1. **Store 注册**: 使用 `Alpine.data(name, data)` 注册响应式数据
2. **x-data 绑定**: 自动设置 `x-data` 属性到视图容器
3. **数据访问**: 通过 `$store.kylin` 访问全局 store
4. **自动清理**: 切换视图时自动清理未使用的 Alpine.data

### 缓存策略

1. **keepalive = true**:
   - 视图存储在 `Map` 中
   - 切换时隐藏/显示（不销毁）
   - Alpine.data 保持活跃
   - 适用于需要保持状态的页面

2. **keepalive = false**:
   - 切换时销毁旧视图
   - 清理对应的 Alpine.data
   - 节省内存
   - 适用于不需要保持状态的页面
